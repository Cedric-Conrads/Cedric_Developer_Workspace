Nach einer Änderung am Projekt:

1.
git status

↓

Prüfen, welche Dateien geändert wurden.

2.
git add .

↓

Die Änderungen für den nächsten Commit vormerken.

3.
git commit -m "Beschreibung"

↓

Einen neuen lokalen Meilenstein speichern.

4.
git push

↓

Den neuen Commit zu GitHub hochladen.

5.
git status

↓

Kontrollieren, ob lokal noch offene Änderungen vorhanden sind.
