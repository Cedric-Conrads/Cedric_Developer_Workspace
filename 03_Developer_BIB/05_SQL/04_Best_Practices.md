# SQL – Best Practices

- Bei Benutzereingaben Prepared Statements verwenden.
- `UPDATE` und `DELETE` nie unüberlegt ohne `WHERE` ausführen.
- Eindeutige Primärschlüssel nutzen.
- Sinnvolle Tabellen- und Spaltennamen wählen.
- Nur benötigte Spalten auswählen, wenn das Projekt größer wird.
- Abfragen zuerst mit Testdaten prüfen.
- Passwörter niemals im Klartext speichern.
